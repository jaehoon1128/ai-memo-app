# 📝 PRD: **Memora** – AI 음성 메모장 웹서비스

---

## 1. Overview

**제품명:** Memora
**제품 유형:** AI 기반 웹 메모 애플리케이션
**서비스 형태:** 웹 애플리케이션 (Web-based Service)

### Problem Statement
기존 메모 서비스는 사용자의 메모를 단순 저장하는 데 그치는 경우가 많아, 시간이 지날수록 중요한 정보를 찾기 어렵고 효율적인 활용이 어려워집니다. 특히 회의록, 아이디어, 공부 내용 등 다양한 형태의 메모를 빠르게 작성하고 구조화하는 것이 점점 더 중요해지고 있습니다.

### Solution
**Memora**는 **AI 메모장 웹서비스**로, 사용자의 음성 또는 텍스트 입력을 받아 **AI가 자동으로 요약 및 분류**해주는 기능을 제공합니다. **STT(Speech-to-Text) 기술**을 통해 빠른 음성 메모 작성이 가능하며, 모든 데이터는 **웹 애플리케이션의 데이터베이스에 저장**됩니다.

### Target Users
- **직장인/프리랜서**: 회의나 작업 중 웹에서 빠르게 메모하고, 자동 정리되길 원함
- **대학생**: 노트북을 통해 수업 내용을 음성/텍스트로 기록하고 정리하길 원함  
- **일반 사용자**: 웹에서 손쉽게 일상 메모를 작성하고 필요할 때 검색하길 원함

### Value Proposition
- **AI 요약**: 긴 메모를 간결한 핵심 요약으로 제공
- **카테고리 분류**: 자동 분류 기능을 통한 체계적인 메모 정리
- **STT 기반 음성 입력**: 빠른 음성 기반 메모 작성 가능
- **웹 접근성**: 설치 없이 브라우저에서 사용 가능

## 2. Core Features

### 2.1 텍스트 메모 작성
- **기능**: 사용자가 직접 웹에서 텍스트 입력으로 메모 작성
- **중요성**: 기본적인 메모 작성 방식으로 모든 사용자가 활용 가능
- **동작 방식**: React 기반 텍스트 에디터, 실시간 저장, 마크다운 지원

### 2.2 음성 입력 (STT 변환)
- **기능**: 마이크 입력을 실시간 텍스트로 변환
- **중요성**: 빠른 메모 작성과 접근성 향상
- **동작 방식**: Web Speech API 또는 Google Cloud Speech-to-Text API 활용

### 2.3 AI 요약
- **기능**: 긴 메모 내용을 간결하게 요약
- **중요성**: 핵심 정보 빠른 파악과 메모 효율성 증대
- **동작 방식**: OpenAI GPT API 또는 Claude API를 활용한 텍스트 요약

### 2.4 자동 카테고리 분류
- **기능**: 메모 내용을 주제 기반으로 자동 분류
- **중요성**: 체계적인 메모 관리와 검색 효율성
- **동작 방식**: AI 모델을 활용한 텍스트 분류 (회의록, 아이디어, 학습노트 등)

### 2.5 검색 기능
- **기능**: 키워드, 작성일, 카테고리별 검색
- **중요성**: 저장된 메모의 빠른 검색과 접근
- **동작 방식**: Elasticsearch 또는 PostgreSQL full-text search

### 2.6 데이터베이스 저장
- **기능**: 모든 메모는 서버의 DB에 저장
- **중요성**: 데이터 영속성과 동기화
- **동작 방식**: PostgreSQL 데이터베이스, Prisma ORM 활용

## 3. User Experience

### 3.1 User Personas

#### Persona 1: 김회의 (회사원, 32세)
- **목표**: 회의 중 빠른 메모 작성과 후속 액션 아이템 정리
- **페인포인트**: 긴 회의 내용을 나중에 정리하기 어려움
- **사용 시나리오**: 회의 중 음성으로 메모 → AI 요약 확인 → 카테고리별 정리

#### Persona 2: 박학생 (대학생, 21세)
- **목표**: 수업 내용을 효율적으로 기록하고 복습
- **페인포인트**: 필기 속도가 느리고 중요한 내용을 놓치기 쉬움
- **사용 시나리오**: 수업 중 음성 녹음 → 텍스트 변환 → AI 요약으로 핵심 정리

#### Persona 3: 이창작 (프리랜서, 28세)
- **목표**: 아이디어를 빠르게 기록하고 체계적으로 관리
- **페인포인트**: 좋은 아이디어를 놓치거나 정리하지 못함
- **사용 시나리오**: 아이디어 음성 메모 → 자동 분류 → 검색으로 빠른 접근

### 3.2 Key User Flows

#### Flow 1: 음성 메모 작성
1. 웹 앱 접속 및 로그인
2. '새 메모' 버튼 클릭
3. 음성 녹음 시작 (마이크 권한 허용)
4. 실시간 STT 변환으로 텍스트 확인
5. 녹음 완료 후 메모 저장
6. AI 자동 요약 및 카테고리 분류
7. 결과 확인 및 필요시 수정

#### Flow 2: 텍스트 메모 작성
1. 웹 앱 접속 및 로그인
2. '새 메모' 버튼 클릭
3. 텍스트 에디터에서 직접 입력
4. 실시간 저장
5. 저장 시 AI 요약 및 분류
6. 결과 확인

#### Flow 3: 메모 검색 및 관리
1. 메인 대시보드에서 메모 목록 확인
2. 검색 필터 적용 (날짜, 카테고리, 키워드)
3. 원하는 메모 선택 및 상세 보기
4. 필요시 편집 또는 삭제

### 3.3 UI/UX Considerations
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 모든 환경 지원
- **직관적 인터페이스**: 최소한의 클릭으로 메모 작성 가능
- **실시간 피드백**: STT 변환 과정의 실시간 텍스트 표시
- **접근성**: 키보드 네비게이션, 스크린 리더 지원
- **다크 모드**: 눈의 피로도 감소를 위한 다크 테마 지원

## 4. Technical Architecture

### 4.1 System Components

#### Frontend (Next.js)
- **프레임워크**: Next.js 14 (App Router)
- **언어**: TypeScript
- **스타일링**: Tailwind CSS
- **상태관리**: Zustand 또는 React Context
- **UI 라이브러리**: shadcn/ui 또는 Radix UI

#### Backend (API Routes)
- **런타임**: Node.js
- **API**: Next.js API Routes
- **인증**: Supabase Auth
- **파일 업로드**: Cloudinary 또는 AWS S3

#### Database
- **주 데이터베이스**: Supabase PostgreSQL
- **ORM**: DrizzleORM
- **검색**: Supabase PostgreSQL full-text search 또는 Elasticsearch

#### External Services
- **STT**: Google Cloud Speech-to-Text 또는 Web Speech API
- **AI 요약**: Anthropic Claude API
- **호스팅**: Vercel

### 4.2 Data Models

#### User Model
```typescript
// DrizzleORM 스키마
export const users = pgTable('users', {
  id: text('id').primaryKey().$defaultFn(() => createId()),
  email: text('email').notNull().unique(),
  name: text('name'),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
});

export const usersRelations = relations(users, ({ many }) => ({
  memos: many(memos),
}));
```

#### Memo Model
```typescript
export const memos = pgTable('memos', {
  id: text('id').primaryKey().$defaultFn(() => createId()),
  title: text('title'),
  content: text('content').notNull(),
  summary: text('summary'),
  category: text('category'),
  audioUrl: text('audio_url'),
  userId: text('user_id').notNull().references(() => users.id),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
});

export const memosRelations = relations(memos, ({ one, many }) => ({
  user: one(users, {
    fields: [memos.userId],
    references: [users.id],
  }),
  tags: many(memoTags),
}));
```

#### Tag Model
```typescript
export const tags = pgTable('tags', {
  id: text('id').primaryKey().$defaultFn(() => createId()),
  name: text('name').notNull().unique(),
});

export const memoTags = pgTable('memo_tags', {
  memoId: text('memo_id').notNull().references(() => memos.id),
  tagId: text('tag_id').notNull().references(() => tags.id),
});

export const tagsRelations = relations(tags, ({ many }) => ({
  memos: many(memoTags),
}));
```

### 4.3 APIs and Integrations

#### Internal APIs
- `POST /api/memos` - 메모 생성
- `GET /api/memos` - 메모 목록 조회
- `PUT /api/memos/[id]` - 메모 수정
- `DELETE /api/memos/[id]` - 메모 삭제
- `POST /api/memos/[id]/summarize` - AI 요약 생성
- `POST /api/memos/[id]/classify` - 카테고리 분류

#### External APIs
- **Google Cloud Speech-to-Text**: 음성 텍스트 변환
- **Anthropic Claude**: 텍스트 요약 및 분류
- **Cloudinary**: 오디오 파일 저장

### 4.4 Infrastructure Requirements
- **호스팅**: Vercel (Frontend + API Routes)
- **데이터베이스**: Supabase 또는 PlanetScale
- **파일 스토리지**: Cloudinary 또는 AWS S3
- **CDN**: Vercel Edge Network
- **모니터링**: Vercel Analytics

## 7. Risks and Mitigations

### Technical Challenges

#### Risk 1: STT 정확도 문제
- **위험**: 한국어 음성 인식 정확도가 낮을 수 있음
- **완화책**: 
  - Google Cloud STT의 한국어 모델 활용
  - 사용자 편집 기능 제공
  - 음성 품질 가이드라인 제공

#### Risk 2: AI API 비용 증가
- **위험**: 사용량 증가에 따른 API 비용 급증
- **완화책**:
  - 요약 길이 제한 설정
  - 캐싱 전략 구현
  - 사용량 기반 요금제 도입

#### Risk 3: 실시간 음성 처리 성능
- **위험**: 긴 음성 녹음 시 브라우저 성능 저하
- **완화책**:
  - 청크 단위 처리
  - Web Workers 활용
  - 최적화된 오디오 포맷 사용

### MVP Scope Management

#### Risk 4: 기능 범위 확장
- **위험**: 개발 중 추가 기능 요구로 인한 일정 지연
- **완화책**:
  - 명확한 MVP 정의
  - 기능 우선순위 매트릭스
  - 반복적 배포로 피드백 수집

#### Risk 5: 사용자 채택도
- **위험**: 초기 사용자 부족으로 피드백 부족
- **완화책**:
  - 베타 테스터 모집
  - 사용자 온보딩 최적화
  - A/B 테스트로 UX 개선

### Resource Constraints

#### Risk 6: 개발 리소스 부족
- **위험**: 복잡한 기능 구현에 필요한 시간/인력 부족
- **완화책**:
  - 단계적 개발 접근
  - 외부 라이브러리 활용
  - 명확한 기술 스택 선택

## 8. Appendix

### 8.1 Research Findings

#### STT 기술 비교
- **Web Speech API**: 무료, 브라우저 내장, 정확도 보통
- **Google Cloud STT**: 유료, 높은 정확도, 한국어 지원 우수
- **Azure Speech**: 유료, 다양한 언어 지원, 커스터마이징 가능

#### AI 요약 모델 비교
- **Claude 3.5 Sonnet**: 높은 품질, 적절한 비용
- **Claude 3 Haiku**: 빠른 응답, 낮은 비용
- **Claude 3 Opus**: 최고 품질, 높은 비용

### 8.2 Technical Specifications

#### 성능 요구사항
- **페이지 로딩**: 3초 이내
- **음성 변환**: 실시간 (1초 이내 지연)
- **AI 요약**: 10초 이내 응답
- **동시 사용자**: 100명 이상 지원

#### 보안 요구사항
- **데이터 암호화**: 전송/저장 시 AES-256
- **인증**: JWT 토큰 기반
- **권한 관리**: 사용자별 데이터 격리
- **API 보안**: Rate limiting, CORS 설정

#### 접근성 요구사항
- **WCAG 2.1 AA 준수**
- **키보드 네비게이션 지원**
- **스크린 리더 호환성**
- **고대비 모드 지원**

### 8.3 Success Metrics
- **사용자 참여도**: 일일 활성 사용자 (DAU)
- **기능 사용률**: 음성 메모 vs 텍스트 메모 비율
- **사용자 만족도**: NPS 점수
- **기술적 성능**: API 응답 시간, 오류율
- **비즈니스 지표**: 사용자 유지율, 월간 활성 사용자 (MAU)

## ✅ 결론 및 향후 과제

Memora는 브라우저 기반에서 음성과 텍스트로 빠르게 메모하고, AI가 자동으로 요약 및 분류해주는 웹 메모장입니다. 

**초기 버전에서는 핵심 기능인 STT 기반 음성 메모, AI 요약, 카테고리 분류에 집중**하고, 사용자 피드백을 바탕으로 검색 고도화 및 시각화 통계 기능을 순차적으로 확장할 예정입니다.

**개발 우선순위**:
1. 기본 CRUD 기능 (Phase 1)
2. AI 요약 및 분류 (Phase 2)  
3. 음성 입력 기능 (Phase 3)
4. 고급 기능 및 UX 개선 (Phase 4)

각 단계별로 사용 가능한 제품을 배포하여 지속적인 피드백을 수집하고, 사용자 요구사항에 맞춰 기능을 발전시켜 나갈 것입니다.
